For this project you will start with an application that can read a .jpg file and convert it into a 2-dimensional array of pixels. The given code allows a user to flip the image horizontally, and to reset the image back to the original version.

You are to add several methods and buttons to the applet. Each method and its button enables one additional picture editing function, such as flip vertical, brighten, darken, convert to greyscale, etc.

  2. Try out the existing functionality.

    1. Run the program in its current form to make sure everything works before making your changes. You should be able to select an image from the drop-down list, then use the buttons on the right to manipulate the image.

    2. The only two buttons/functions that are implemented are flip horizontally, and restore the image to its original state.

    3. We have provided six image files, but you may add .jpg files of your own if you wish. To do so, copy your .jpg file(s) to the main project folder (NOT the src sub-folder), then add a line of code in the MUPanel constructor method to add your .jpg file's name to the Choice control.

  3. Look over the source code to see how the program is structured and generally how it works.

    1. The Pixel class represents one pixel of an image. Think of a Pixel as an (over, down) coordinate. Each Pixel has an amount of red, green, and blue.

    2. The Picture class represents an image as a two-dimensional arry of Pixels (rows and columns of Pixels).

    3. We display the picture by using a large Label, and putting an Icon in the Label (the Icon contains the image).

    4. Look over the way we used an array of buttons. We start with only two "active" buttons that actually do something (Flip Horizonally & Restore).

    5. To add a new function, you'll need three steps:

      • Add a method to the application (for example doFlipVertically).
      • Add two new lines to the "createButtons" method.  The new lines should allocate a new Button with a label that describes the new functionality you are adding to the application, and then "addActionListener(this)" for the new button.  All of the buttons should be in the array named "buttons" -- use a new number for each button you add.
      • Add a new "else if" block to the code in the actionPerformed method.  If "source" is equal to buttons[x] (where x is the number of the new button you have added), then call your new "do" method to actually perform the image manipulation, and then call "repaint()".

  4. Figure out how flip horizontally works, and then add the functionality to flip vertically.

    1. Each method you add to the applet must have three parts:

      • Declare a two-dimensional array of Pixels, and use currentPicture.getThePixels() to assign it a value.
      • The middle part where you manipulate/change in some way the two-dimensional array of Pixels.
      • Use currentPicture.setThePixels to put your changed version of the Pixels/image back into the picture.

    2. Study the doFlipHoriz method to figure out how it works. Then use this as a guide to write a doFlipVert method.

    3. The doFlipHoriz method uses a nested for loop to "walk through" all the rows & columns of Pixels.

    4. For each pixel we swap it's value with the value of the corresponding pixel in the same amount from the right as this pixel is from the left.

  5. You must do each of these additional functions.

    1. Remove Red. Use a nested for loop to "walk through" all the rows & columns of Pixels. For each Pixel set its red amount to 0. (Look inside the Pixel class and find that there is a method called setRed.

    2. Greyscale. For each Pixel calculate the average of its red, green, and blue amounts (using getRed, getGreen, and getBlue). Then change its amount of red, green, and blue to that amount (using setRed, setGreen, setBlue).

    3. Lighten. For each Pixel get its red amount. If amount of red is less than or equal to 235, then add 20 to it. In other words set it to (20 + what-it-was). Otherwise (else) set the amount of red to 255. (You'll need to use getRed, and also setRed.) Now do the same thing with green, and the same again with blue.

    4. Darken. Similar to lighten, except subtract 20 from the red, green, and blue of each Pixel or set the amount to 0 if subtracting 20 would make it negative.

  6. You must choose any 1 of these additional functions.

    1. Polarize. Calculate the average red for all pixels. (Add up the red amount for all pixels in all rows and all columns. Then divide by the total number of pixels.) While you are adding up all the reds, at the same time (in same loops) add up the greens, and the blues. Then outside the loops divide each total by the total number of pixels so you have an overall average amount of red, average amount of green, and average amount of blue. Now use another set of nested loops to change each pixel. For each pixel if its red is less than the average amount of red, then set its red to 0, else set its red to 255. Similarly, if its green is less than the average amount of green, then set its green to 0, else to 255. Similarly, handle the blue.

    2. Flip Corners.

    3. Scroll Horizontally. Use some fixed number of pixels (in the 30-80 range) to scroll. You'll need an extra copy of the entire two-dimensional array (one for what the picture was, and one for what you're changing it to). "Move" each pixel to the right by the fixed amount. If a pixel would be moved past the right edge, then "wrap" it around to the left.

    4. Scroll Vertically. Similar to Scroll Horizontally, except scroll downward instead of to the right.

    5. Negative. For each Pixel set each of the red, green, and blue values to (255 - what-it-was).

    6. Blur. To blur an image, you assign each Pixel value the average of all its Pixel neighbors (i.e. its red component gets the average of all red components of Pixels around it, ...). To do this properly, you will need to make a copy of the image. Use the following algorithm for bluring each pixel: 
       
If the copy of your image had a section of Pixels whose Red components
look like this:
 
 +-----------------+
 | 116 | 119 | 65  |
 +-----------------+
 | 85  | 14  | 31  |   
 +-----------------+
 | 177 |  5  | 55  |
 +-----------------+
 
In your blurred image, the new value of the Red component for the
center Pixel would be the average of all of the pixels in the 3x3 square:
 
 +-----------------+
 |     |     |     |
 +-----------------+
 |     | 74  |     | 
 +-----------------+
 |     |     |     |
 +-----------------+
      Be careful at the edges of the image: not all pixels have 8 neighboring pixels!  Note: to really see what this is doing, repeatedly click on the Blur button many times to make the image progressively blurry; then click "Restore Image" to get back to sharp clarity.
